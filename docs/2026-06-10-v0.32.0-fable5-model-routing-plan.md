## Context

Claude Fable 5 (`claude-fable-5`, released 2026-06-09, $10/$50 per MTok — see
`docs/KB-fable-5.md`) is the top-tier Claude model, above Opus 4.8 ($5/$25);
Haiku 4.5 is **10× cheaper** than Fable on both axes. The plugin last shipped
(v0.31.1) when Opus was the ceiling. Two problems:

1. **No Fable awareness.** Model-tier references across skills/statusline/policy
   assume Opus = top.
2. **All-Fable swarms.** Agent-tool model tokens are `sonnet | opus | haiku |
   fable` (verified live; probe record in Step 0). `model: "opus"` does NOT
   resolve to Fable. A spawn that omits `model` inherits the orchestrator's
   model. A Fable orchestrator that doesn't set `model` per spawn gets an
   all-Fable swarm. Nothing enforces routing: `plugins/anti-hall/hooks/
   swarm-guard.js` discards stdin (:228), checks only spawn rate + memory.

v0.32.0 adds the Fable tier, makes routing enforceable (default-advisory +
opt-in strict), upgrades the debate roster to a cross-model TRIO, and adds a
Workflow-based "deadly swarm" execution mode.

## Workstream B — `hooks/model-routing-guard.js`

### Contract
- `hooks.json`: FIRST in BOTH Agent and Task arrays (before swarm-guard +
  phase-tracker) — a routing block must not leave swarm-spawns.log /
  phase-tracker entries. Defensive: host may run all matcher hooks anyway;
  ordering minimizes recorded state; a block/retry cycle costs ≤1 extra
  rate-cap entry (accepted). Order asserted by test (spawn-hook.js harness
  reads hooks.json — confirmed implementable, A2 cleared).
- stdin parsed against Step-0 P3 fixtures. Pure Node, cross-platform,
  fail-open on ANY error, skip hatch (`isSkipped('model-routing-guard')`),
  `fs.writeSync(1, …)`. 128 KB scan bound; `typeof === 'string'` guards.

### Classification
- NFKC + casefold + word-boundary token matching.
- **Debate-role exemption — short metadata fields ONLY, suppresses row-1 BLOCK only**
  (advisories still emitted — R2-4; narrowed from prompt-wide per C2-3):
  `/\b(reviewer|auditor|critic|debate|deadly[- ]?loop)\b/i` against
  `tool_input.description` AND, if P3 shows it arrives as a distinct field,
  the workflow `label` (A3-2: label→description mapping is UNPROVEN until P3).
  Independent of that mapping, A4 adds an explicit role-word `description` to
  every shipped Agent-tool trio snippet (guaranteed plumbing); workflow.js
  spawns get `description` IF P5 shows `agent()` accepts it, else their
  role-worded `label` is the documented fallback path (R4-2 — claim scoped
  honestly). Prompt-body role words do NOT exempt.
- Mechanical signals (→ haiku): fetch/download/curl, grep/glob/search files,
  run command/script, install, build, run tests (execution-only), git push,
  deploy, dump/export, tail/read logs, list/check status.
- Complex signals: plan/planning, design, architect, review, audit, regression,
  coupling, merge order, critique, debate, validate, simulate, root cause,
  workflow analysis, logic, mockup, security. Complex-anywhere ⇒ never block.

Model matching note (R3-2/C4-2 simplified): row 1 matches the Agent-tool enum

- Advisory channel: nested `hookSpecificOutput.additionalContext` (Step-0 P2).
- Block reason does NOT advertise the skip hatch (R-4).

### Threat model (honest — C2-2)
- The ORIGINAL reported bug (Fable orchestrator omitting `model` on swarms) is
  mechanically blocked only in **strict mode**; default mode mitigates it via
  row-2 advisory + the A3 always-set-model rule. Default is advisory because
  the hook cannot see the parent model and cheap-orchestrator false blocks are
  worse (R-1). Strict mode is for users who routinely orchestrate on flagship.
- **Strict-mode trade-off (replaces the withdrawn heuristic — A4-1):** strict
  blocks EVERY omitted-model mechanical spawn, including on a genuinely cheap
  orchestrator. Opt-in accepts this; remedy in the block text (set an explicit
  model, or unset strict). Documented in hook header + README row.
- **Failed P2 re-probe / older harness (A4-2, C4-4, stated honestly):** the
  exemption-advisories and rows 2-default/3/4 deliver ONLY via additionalContext. If the implementation-day
  re-probe fails, the guard ships as row-1 (+ strict row-2) BLOCK-only and the
  advisory rows are SILENT NO-OPS — the advisory surface vanishes; the plan does
  not pretend advice survives via another channel. On a user harness that does
  not deliver PreToolUse additionalContext, the same degradation applies at
  runtime (fail-open, nothing breaks, advisories inert) — documented limitation;
  a hook cannot enforce a harness min-version.
- Worst failure = wrong-BLOCK → explicit-model row, exemption modifier
  (row 1), complex-anywhere veto, strict = opt-in + project-scoped, fail-open.
- Accepted failure = waste-ALLOW (keyword stuffing evades; C-3): anti-waste
  net, not a security boundary — stated in hook header.

### Round governance
- **Flaky seat:** one retry per seat per round. RETRY ARITHMETIC (A2-2): retries
  fire inside the same 60s window as the round dispatch — quadruple-multi worst
  case 12+12=24 > swarm-guard's 20-cap. Rule: at multiplier ≥ triple, retries
  are SEQUENTIAL after the wave settles, or accept the documented +1-entry
  blocked-retry cost; state in deadly-loop-multi.
- **DEGRADED round** (a seat dead after retry): proceeds with 2 verdicts for
  iteration purposes, but **a DEGRADED round can NEVER grant final GO**
  (R2-7/C2-7) — the missing seat must sit in a full follow-up round.
- **Dissent:** GO requires zero un-adjudicated HOLD blockers. Single-seat re-run
  is allowed ONLY for evidence-adjudication with NO code/plan change in between
  (R2-1/C2-8); ANY fix wave ⇒ full trio next round (preserves SKILL.md:392
  anti-pattern). Evidence-refuted dissent may be overridden, documented in
  handoff. ≥2-of-3 agreement = confirmed-real.

## Workstream E — Deadly SWARM (Workflow execution mode)

`skills/deadly-loop/references/deadly-loop.workflow.js` (precedent:
ship-it.workflow.js), gated on Step-0 P4/P5:

- **Scope: ONE workflow run = ONE round + its fix wave** (C2-4/C2-5/R2-6). The
  COORDINATOR loops: run workflow → read verdicts → update handoff doc (durable
  state lives in the handoff, not workflow memory) → decide → next run.
  Soft-cap-10 checkpoint happens at coordinator level via AskUserQuestion
  BETWEEN runs (workflows cannot prompt mid-run); hard-15 enforced by the
  coordinator and double-guarded by a round-counter arg the script refuses past 15.
- Trio seats: `agent(brief, {model:"fable"})` / `{model:"opus"}` /
  `{agentType:"codex:codex-rescue"}` per A1 canon. Script begins with the
  codex availability probe (MODEL-POLICY Node probe) and seats the Opus
  adversarial fallback if absent (C2-6).
- Synthesis: `VERDICT_SCHEMA` (structured output) with canonical dedup key
  `{file, category, lineBucket}` where lineBucket = floor(line/20), null for
  line-less findings (R4-4); collisions are MERGE-PRESERVING — same-key
  findings grouped into an array, never collapsed; SCOPE STATED HONESTLY
  (C4-3): only same-bucket collisions auto-group — adjacent cross-bucket
  near-duplicates (e.g. lines 19/21) reach the coordinator as separate entries
  for adjudication; the script does not promise semantic dedup. Free-text
  residue goes to the coordinator, not dropped.
- Operator friction stated (C3-5): each Workflow call may require harness
  consent unless pre-allowed; a 7-round loop = up to 7 prompts. The SKILL
  documents this and the skill-invocation opt-in path.
- No cross-round caching (A3-5): each round = fresh workflow process; durable
  state is the handoff doc ONLY. The SKILL must not imply resume across rounds.
- Determinism header copied from ship-it.workflow.js:31-32 (no Date.now /
  Math.random / argless new Date); timestamps passed via `args`; static test
  greps the script for forbidden APIs (C2-10).
- deadly-loop-multi N-trio = parameter of the same script.
- SKILL.md documents: swarm mode (multi rounds/rosters) vs inline mode (small
  audits); Workflow harness opt-in requirement; guard-coverage boundary from P4.

### E.1 — 3-phase swarm architecture (owner-approved upgrade; ALL loops swarm-first)

**Phase 1 — CONTEXT AGENT** (`agent(contextBrief, {model:"sonnet"})`, first call):
- round 1 (initial): comprehensive pack — target files, prior history (handoff
  path via args), scope declaration, line anchors. Graphify freshness check
  HERE ONLY (update if stale; measured ~7 min — never per-round).
- round >1 (incremental): prior pack (file written by previous Phase 1) +
  findings + fixes applied + reinvestigation instructions; does NOT re-read
  targets in full. Output = context-pack FILE (path in structured output) —
  seats' shared source; kills the measured 3× redundant re-reading. Fresh pack
  per round (prior pack is input, not cache — A3-5 preserved).

**Phase 2 — THE DUEL (Option A — SELECTED):** 2a independent investigation
(trio, parallel, context pack only) → 2b STRUCTURED ARGUMENT: each seat
receives the other seats' finding arrays and responds per finding with
`refute` (file:line counter-evidence) / `corroborate` / `escalate`. Two
parallel() waves; Codex participates identically (staged file I/O, no live
channel). ~+3 agent() calls/round. (Option B — live cross-talk via shared
polling file — DEFERRED: Codex mid-run shared-file I/O unverified; revisit
only with a dedicated probe. Independence of 2a is load-bearing: Rounds 4-5's
P0s came from non-anchored divergent seats.)

**Phase 3 — CONVERGE & CONFIRM:** VERDICT_SCHEMA dedup as specified; 2b
responses feed adjudication (≥2-of-3 + argument outcomes). **RESPAWN-ON-DRIFT
— OBJECTIVE criteria ONLY** (dissent is NEVER drift; dissents go to 2b):
(1) branch/SHA mismatch vs targetSHA arg; (2) wrong scope audited;
(3) schema-required file:line evidence missing; (4) VERDICT_SCHEMA parse
failure. Respawned seat gets corrected pack + drift-reason code + the
offending output — NOT peer findings (contamination). Respawn counts against
the one-retry-per-seat quota.

**Wrapper + fallback:** deadly-loop SKILL = thin wrapper invoking the
workflow; deadly-loop-multi = multiplier param; ship-it gates call it. Plain
Agent-tool path stays DOCUMENTED (not deprecated) for no-Workflow-consent
sessions: coordinator spawns trio directly, passes pack path in briefs.
