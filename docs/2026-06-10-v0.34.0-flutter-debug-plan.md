## Workstream D — The debug loop (agent body; mirrors KB §5 + root-cause discipline)

0. **Preflight** (C) — pick capability tier; announce it.
1. **Run:** user (or launch_app if FP4-verified enabled [2]) starts `flutter run --print-dtd`; connect via `dtd` →
   `listDtdUris` → `connect` [2].
2. **Reproduce:** PRIMARY: marionette `tap`/`enter_text`/`scroll_to` by widget key/text [5]; OS-level surfaces or
   no-package fallback: `ui_describe_all`/`ui_find_element` then `ui_tap`/`ui_swipe`/`ui_type` [10]. Screenshot BEFORE
   state [5][10]. **No-repro ⇒ no fix** (root-cause skill).
3. **Read the error — routing:** exceptions → `get_runtime_errors` [2]; layout → `widget_inspector` [2]; prints/logs →
   marionette `get_logs` (needs LogCollector [5]), or `get_app_logs` — **double-gated: ONLY on the launch_app-enabled
   path (FP4-verified), NOT default manual-run; skip when lifecycle disabled** [2] (NEW-1) — else read the user's
   `flutter run` terminal; static → `analyze_files`/lsp [2]; protocol → `call_vm_service_method` [2].
4. **Root-cause (anti-hall `root-cause`, NO CAUSE NO FIX):** prove the mechanism from error + widget tree + code first.
5. **Fix:** edit (native tools); verify statically with `analyze_files` [2].
6. **Reload:** `hot_reload` (state preserved); `hot_restart` for const/initializer/state-reset [2].
7. **MANDATORY re-verify:** re-run `get_runtime_errors` until clean (the official demo loop does this re-check [1]);
   with a screenshot tool, capture AFTER vs the repro — otherwise report "error-clear but visually unverified", never
   "verified".
8. **Escalation trigger (agent body — R1-F5/A1-F5/C9):** after 2 full loop iterations without a proven root cause, or
   when the fix needs architecture redesign, STOP and report `escalate: opus` with collected evidence — the agent
   never respawns itself.
9. Loop 2–7 until clean; report = capability tier + per-fix evidence (errors before/after, screenshots).
